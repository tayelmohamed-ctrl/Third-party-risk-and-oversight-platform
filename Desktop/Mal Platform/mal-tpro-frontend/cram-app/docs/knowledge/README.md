# Knowledge base — authoritative logic

These three skills encode the full CRAM requirements. They are the source of truth
for *logic*; the `docs/` specs and `seed/` data operationalise them for the build.

- `cram-risk-methodology/` — how risk is scored, overridden and routed (model logic).
- `cram-implementation-brd/` — what must be built, tested and accepted (requirements).
- `cram-technical-implementation/` — how to build it: formula, schema, APIs, pseudocode, SIT/UAT.

When a build doc says "see the methodology skill", read the matching SKILL.md and
its `references/` files here. Keep these in sync with any model-version change.
